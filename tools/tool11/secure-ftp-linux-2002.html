Secure FTP Tool - Linux Implementation (2002)

Year: 2002 | Language: C | Platform: Linux | Security: RSA Encryption

A college project that implemented a secure file transfer protocol with client-server architecture, user authentication, and RSA encryption for secure data transmission.

Project Overview

In 2002, during my college days, I developed a secure FTP tool in Linux using C programming language. This project demonstrated the fundamentals of network programming, cryptography, and secure file transfer protocols. The tool featured a client-server architecture with RSA encryption for secure data transmission.

RSA Encryption Implementation

The tool implements a complete RSA encryption system from scratch, including prime number generation, key derivation, and encryption/decryption algorithms.

Prime Number Generation

primegen(double *p, double *q) { int i, m, prime, it; int flag = 0; int primes[5000], len = 0; for(m = 3; m < 2000; m++) { for(i = 2; i < m - 1; i++) { if((m % i) == 0) { flag = 0; break; } else { flag = 1; } } if(flag == 1) { primes[len++] = m; } } m = 1.0+(int)((double)len*rand()/(RAND_MAX+1.0)); *p = primes[m]; it = 1.0+(int)((double)len*rand()/(RAND_MAX+1.0)); *q = primes[it]; }

Key Generation

// Public key calculation publickey(double p, double q, double *n) { *n = p * q; return; } // Private key calculation cal_d(double e, double phi, double *d) { double n = 0.0; for(n = 1.0; n < phi; n = n + 1.0) { *d = 1.0+n*phi; *d = fmod(*d, e); if(*d == 0.0) { *d = (1.0+n*phi)/e; return; } } }

Encryption/Decryption

encrypt(double e, double n, double *plain, double *cipher) { double cip, M, tem = 0.0; M = pow(*plain, e); cip = fmod(M, (float)n); *cipher = cip; } decrypt(double d, double n, double *cipher, double *plain) { double pla, M, tem = 0.0; M = pow(*cipher, d); pla = fmod(M, (float)n); *plain = pla; }

Protocol Commands

  • cwd: Show current working directory
  • ls: List remote files
  • cd: Change remote directory
  • get: Download file from server
  • put: Upload file to server
  • lls: List local files
  • lcd: Change local directory
  • bye/exit: Terminate connection

Conclusion: Learning Through Implementation

Building a secure FTP tool from scratch in 2002 was an excellent learning experience that covered multiple aspects of computer science and cybersecurity. This project demonstrated the importance of understanding low-level concepts like network programming, cryptography, and system programming.

The implementation showed that security is not just about using existing tools, but understanding how they work at a fundamental level. By implementing RSA encryption, TCP communication, and user authentication from scratch, I gained deep insights into how secure systems are built.

This project remains relevant today as it teaches the fundamentals of secure communication protocols, cryptography implementation, and network security - skills that are essential for modern cybersecurity professionals.
