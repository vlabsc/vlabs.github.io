Before Defender: A Look Back at MSAV and CPAV

If you think Microsoft Defender has always been there quietly protecting your computer, let me take you back to the time when we had to run antivirus tools manually in DOS. Files were scanned one by one on those big CRT monitors.

I am talking about MSAV and CPAV — two tools that helped us fight boot-sector viruses and file infections. I recently tried running MSAV.EXE and CPAV.EXE using DOSBox, just to bring back those old memories. And it really felt like going back in time.

The Simpler Days of CPAV

CPAV (Central Point Anti-Virus) was an antivirus tool used in DOS during the early 90s. It was made by a company called Central Point Software — the same people who created the well-known PC Tools.

The CPAV files were usually arranged like this:

In those days, CPAV did not run in the background like today’s antivirus. You had to open it manually by typing something like:

A:\> CPAV.EXE

Once you run it, you will see a colourful text screen with menu options, scan settings, and alerts. First, it checks if any virus is already in the memory.

The best part was, you could see the files getting scanned one by one. On one side, the file names would scroll, and below that, a small bar would keep moving. It looked like the virus was getting caught right in front of you.

Once the scan is completed, a summary of files and directories scanned is displayed.

Look how the configuration files were kept .. The CPAV.ini file…

The CPAV.INI file was used to store the configuration settings for CPAV. It was a simple text file that contained the settings for the antivirus program.

The menus and Options were exhaustive and look at the way the menu bars and options were designed during 90s.

From CPAV to MSAV — Microsoft Enters the Game

Later, Microsoft took CPAV and used it as their own antivirus. They renamed it as MSAV (Microsoft Anti-Virus) and included it with MS-DOS 6. So no need to install it separately.

You could open it by typing:

C:\> MSAV

It looked almost the same as CPAV, just with Microsoft’s name on it.

Look at the similarity between MSAV and CPAV (image source Wikipedia)

MSAV had a few useful options that were easy to use.

  • Scan memory — It first checked if any virus was already loaded in the system memory.
  • Check boot sector — It could check the boot sector of floppy or hard disk, where many old viruses used to hide.
  • Scan full hard disk or floppies — You could scan the full drive or just the floppy disk to see if any file was infected.
  • Schedule scan — You could set it to scan at a particular time automatically.
  • Load VSAFE.EXE — This was a small program that stayed in memory and kept watching in the background. It showed alerts if any program tried to do something suspicious.

    • What felt cool back then

    Those days, viruses like Stoned, Michelangelo, and Form were very common. Running MSAV felt serious, like doing some urgent work. When it showed “No virus found,” we used to feel relaxed.

    And when VSAFE gave a warning while opening a .COM or .EXE file, it looked advanced for that time — like the computer was really watching.

    So What Happened to CPAV?

    In 1994, Symantec (the company behind Norton) bought Central Point Software. After that, CPAV slowly disappeared, and many of its features were added into Norton Antivirus.

    MSAV also stopped coming in later Windows versions, as most people started using antivirus software with proper menus and mouse support. Norton and McAfee became more popular during that time.

    Microsoft came back into antivirus only in 2005, after big viruses like Blaster and Sasser caused problems. That’s when they started Windows Defender.

    Final Thoughts

    Things have changed a lot now. Earlier, we used to type MSAV in a black-and-white screen. Now, Defender works on its own using AI and cloud scanning, and we do not even notice it.

    But those old tools had a different feel. You could see everything. You ran the scan yourself. You selected the floppy drive. You pressed “Y” to remove a virus. It was not automatic — you were fully involved in protecting the system.

    CPAV’s Strengths — Ahead of Its Time

    Central Point Anti-Virus (CPAV) was not just a basic scanner, it was a seriously smart tool for its era. Here’s what made it stand out in the early 90s:

  • Real-time protection using TSR (CPAVTSR.EXE)

    • CPAV could stay in memory and keep watching quietly. If any file was changed, opened, or if something touched the boot sector, it would alert. This kind of real-time checking was not common before Windows 95.

  • Heuristic checking

    • Even if the virus name was not known, CPAV could catch it by looking at suspicious behaviour — like if a file tried to copy itself or change code. This was an early method of checking without depending only on known virus names.

  • Boot sector and memory scanning

    • It could check the boot sector of floppy and hard disk — many viruses like Stoned, Form, or Michelangelo used to hide there. It also scanned the system memory before loading any program.

  • Updates through floppy

    • There was no internet in those days. So we got virus updates through floppy disks — sometimes from computer magazines, or sometimes a friend who had the latest files. These updates came as .DAT or .DEF files.

  • Simple UI with keyboard support

    • Even though it was a DOS program, CPAV had proper menus. You could use mouse also, but it was fully usable with just the keyboard. It was easy for both home users and office staff.

  • Clean-up and report

    • CPAV tried to clean the virus without deleting the file, if possible. It also kept a small log file, so you could check what was scanned or cleaned later.

    The Story Behind Microsoft Licensing CPAV

    Back in the early 90s, Microsoft was preparing MS-DOS 6.0, and antivirus threats were becoming serious, boot sector viruses were spreading via floppies at computer labs, schools, and offices.

    Rather than build an AV engine from scratch, Microsoft licensed a proven tool.

    I vaguely remember what happened, if I am correct (checked with Google), these are the events:

  • 1991 — CPAV gains popularity as part of PC Tools by Central Point Software (CPS)
  • 1992 — CPAV becomes one of the most used AV tools for DOS
  • 1993 — Microsoft licenses CPAV to bundle with MS-DOS 6.0 as MSAV.EXE
  • 1994 — Symantec acquires CPS, merges CPAV tech into Norton AV
  • Post-1994 — Microsoft retires MSAV with the rise of Windows 95

    • How It Was Announced

    The Microsoft-CPAV deal made news in PC Magazine, InfoWorld, and Byte magazines around 1993.

    “Microsoft Anti-Virus, based on Central Point’s trusted scanning engine, offers real-time protection against file and boot sector viruses…”

    — PC Magazine, April 1993

    The product was positioned as “built-in protection for every MS-DOS 6 user.” It was marketed as simple, no-install, no-cost antivirus at a time when Norton and McAfee were paid products.

    Ads and Brochures of the Time

    Most advertisements were bundled in DOS 6.0 retail boxes, not separate AV ads. You’d find messages like:

    “Now with Microsoft Anti-Virus — powered by Central Point technology.”

    CPAV was still available

    Even after MSAV was replaced, CPAV.EXE and VSAFE.EXE continued to circulate unofficially on BBS boards and floppy collections well into the late 90s.

    Tools like CPAV and MSAV may look old and simple, but back then they were quite powerful, and they played a serious role in shaping how we think about digital safety. They were simple, fast, and surprisingly effective for their time.
